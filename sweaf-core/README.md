## Файловая структура кастомной метамодели smlteaf-core

```

|- datasets             - Общие datasets
|  |- datasets.yaml     - Общие datasets используемые в рамках всей метамодели, например, для отчетов
|  |- root.yaml         - В файле осуществляется импорт datasets
|- entities             - Описание кастомных сущностей, а также расширения для дефолтных сущностей. Теперь каждая сущность привязана к домену и полностью автономна. Т.е. её можно вынести из метамодели и использовать отдельно
|  |- app               - Прикладная архитектура
|  |- ba                - Бизнес-архитектура
|  |- ia                - Информационная архитектура
|  |- ta                - Техническая архитектура 
|  |- root.yaml         - Импорт сущностей
|- functions            - Кастомные функции
|  |- functions.yaml    - Общие функции используемые в рамках всей метамодели, например, для отчетов
|  |- root.yaml         - В файле осуществляется импорт функций
|- rules                - Кастомные валидаторы
|  |- accounts.yaml     - Валидация аккаунтов
|  |- systems.yaml      - Валидация параметров системы
|  |- technologies.yaml - Валидация параметров системы
|  |- valid_system.yaml - Попытка сделать валидацию системы через JSON schema
|- README.md            - Данное описание
|  |- root.yaml         - Импорт всех элементов кастомной метамодели, также описана структура пакета кастомной метамодели [archpkg](https://www.npmjs.com/package/archpkg)

```
